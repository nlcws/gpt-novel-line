# PW90_DS90_ARTIFACT_HANDOFF_JOINT_LOCK v001

STATUS: current_writer_core
APPLIES_TO_RUNTIME: pw90-v004.12-nlcore-locked

---

## 0. 上位核

```text
条件内で一切の妥協をせずに、限界まで本文を出す。
```

PW90 はこの核を、ZIP話パック内条件の全回収と本文起こしとして実行する。

---

## 1. 受領基準

PW90 は、設計さんの版番号や梱包工程の実装コードではなく、話パック成果物を受領判定する。

```text
受ける:
  ZIPで梱包されている。
  ZIP内に小説を書ける材料がある。
  その材料から条件を拾って本文化できる。

拒否しない:
  DS90の版番号が違う。
  梱包工程の実装が違う。
  ready / V2 / layer / frozen がない。
  manifest がない。
  最新版棚形式ではない。
```

正規話パック形式は最高形であり、最低受領条件ではない。

---

## 2. WRITE前の責任

PW90 は、話パック内にある条件をすべて拾う台帳を作る。

```text
拾う:
  本文化できる本文材料
  条件
  禁止線
  熱量
  接続
  明示された不足

拾わない:
  話パック外の推測
  版番号からの推測
  ファイル名だけの読了扱い
  summary / memory / filelist だけの読了扱い
```

---

## 3. DS90への尊重

DS90 は、PW90 が限界まで本文を出せるように、作成時点で必要条件を閉じた話パックを渡す。
PW90 は、DS90 が閉じた条件を軽視しない。

```text
DS90 は穴をPW90へ押し付けない。
PW90 は閉じた条件を勝手に値引きしない。
STOPは責任追及ではなく、成果物が最終核を満たせない状態を示す札である。
```

---

## 4. STOP境界

STOPしてよいのは、本文化不能な場合だけである。

```text
ZIP破損
中身が読めない
小説を書く材料が実質ない
条件矛盾で本文化不能
書くなという明示がある
話パック内条件を全回収する台帳が作れない
```
