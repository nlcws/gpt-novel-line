# PW90_EPISODE_TAIL_NEXT_OPENING_BRIDGE_LOCK_v001

## 目的

毎話再起動を復活させるだけでは、前話との接続の手触りが消える。
しかし前話全文・前話文体・前話SUCCESS顔を持ち越すと、NEW_EPISODE_FULL_BOOTが不発になる。

PW90 v004.20 では、各話WRITE単位を次の形に固定する。

```text
CURRENT_EPISODE_FULLBURN
+
NEXT_EPISODE_OPENING_BRIDGE
```

これは「2話ぶんを書く」ではない。
現話を完全に書き切ったあと、次話冒頭へ渡すための最小接続火種を作る。

## 基本原則

```text
前話全文は破棄する。
前話の下部だけを接続アンカーとして読む。
現話をFULLBURNで書く。
現話の末尾から、次話の書き出し候補まで作る。
次話書き出し候補は正本ではなく、次話NEW_EPISODE_FULL_BOOT用の接続札である。
```

## 前話下部確認

対象話を書き始める前に、前話の下部だけを確認する。
ここで読むのは文体・圧縮癖・成功顔ではない。

読むもの:

```text
前話の最後の生活状態
最後に残った物
最後の位置関係
残された問い・札・余韻
前話末尾で未処理のまま残した火種
次話へ自然に渡すべき生活動線
```

禁止:

```text
前話本文の文体を継続する
前話の薄い成功判定を持ち越す
前話の省略癖・圧縮癖を引き継ぐ
前話LOGを現話の本文燃料にする
前話のBODY_LOCALを正規固定として持ち込む
```

## 次話書き出しブリッジ

現話本文と本文後LOGを閉じた後、次話の書き出し候補を作る。
これは現話本文の一部ではない。
現話文字数にも、現話SUCCESSにも、現話coverageにも数えない。

次話書き出し候補に含めるもの:

```text
次話冒頭の最初の生活物
次話冒頭の位置
前話末尾から自然に続く手元・空気・問い
次話が始まる時に消えていないはずの小さな状態
次話話パック再実読時に照合すべきWARN
```

次話書き出し候補は、次話話パックを読む前の仮橋である。
次話話パックと矛盾した場合、必ず仮橋を破棄し、次話話パックを優先する。

## 出力扱い

通常のWRITE出力5点は維持する。
ただし本文後LOGの末尾に、必要なら次を加える。

```text
NEXT_EPISODE_OPENING_BRIDGE:
- status: HANDOFF_ONLY_NOT_CANON
- previous_tail_anchor:
- next_opening_candidate:
- carryover_allowed:
- carryover_denied:
- revalidate_against_next_episode_pack: REQUIRED
```

## 成功条件

現話SUCCESS候補には、以下が必要。

```text
前話下部を読んだが、前話全文惰性を持ち越していない
現話は対象話フォルダを再実読してFULLBURNしている
次話書き出し候補を、正本ではなく接続札として分離している
次話書き出し候補が現話本文量・現話coverageへ混入していない
次話話パック再実読で矛盾した場合に破棄する宣言がある
```

## STOP

次の場合は本文前に止める。

```text
前話下部が読めないのに、強い接続が必要
前話下部から引いた内容が現話話パックと矛盾する
次話書き出し候補を正本扱いしようとしている
次話話パックを読む前に次話本文を確定しようとしている
前話文体惰性を接続と誤認している
```

## 核

```text
毎話、玄関から入り直す。
ただし、前の部屋の最後に置いた湯呑の位置だけは見てから入る。
出る時は、次の部屋のドアノブに触れるところまで書く。
```
