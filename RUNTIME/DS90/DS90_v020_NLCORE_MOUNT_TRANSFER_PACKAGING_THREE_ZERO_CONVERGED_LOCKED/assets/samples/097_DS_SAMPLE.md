## 0. 位置づけ

このファイルは、サンプル、試走、比較例、例示文を隔離する。

## 1. サンプル隔離原則

サンプル内の作品名、人物、店、地名、設定、事件、空気は対象作品の条件ではない。

サンプルは形式確認と動作確認にだけ使う。

対象作品の作品骨、話カード、本文条件、禁止線へ移植しない。

---

## 2. 抽出してよいもの

- 見せる量
- 候補提示の形
- 好き / 違う / もっとの回収方法
- 熱源LOGの取り方
- 条件化の仕方
- EVENT_LIST_TO_CAUSAL_FLOWの見せ方
- 普通の出来事列の受け方
- 出来事列を因果役割へ変換する方法
- GOAL_TO_ANCHOR_FLOWの見せ方
- ゴール、起点、次起点の置き方
- 起点間シミュレーションの窓
- 日次シミュレーションを下位処理にする方法
- 生LOGから接触点を抜く手順
- 候補処理タグ
- プロット化表
- 厚カードへの変換規則

---

## 3. 抽出してはいけないもの

- 作品名
- キャラ名
- 店名
- 世界観
- 関係性
- 固有小物
- サンプル内の事件
- サンプル内の口調
- サンプル内の読後感そのもの
- サンプル由来の禁止線を無検証で他作品へ移すこと

## 4. 初心者向け提示例

ユーザーに最初から設定表を求めない。

ユーザーに「何でも作れます」と言わない。

ユーザーに工程名を押しつけない。

標準提示例。

```text
まず最後に見たい場面を決めます。
次に、普通の出来事列を雑に出してください。
その出来事を、そのままプロットにはしません。
最後に必要な条件へ通るように、こちらで役割へ変換します。
思いつきイベントは入れません。
```

圧縮提示例。

```text
最後に読者へ残したいものを1つ置きます。
次に、普通の出来事列を出します。
こちらで、その出来事をゴールへ向かう役割に変換します。
```

候補提示例。

```text
最後に残すもの候補A:
読者が、この子を名前で呼べるようになる。

最後に残すもの候補B:
最初はただの制度に見えたものが、最後にこの子の居場所へ変わる。

最後に残すもの候補C:
失った逃げ道が、後から必要な因果だったと分かる。
```

回収例。

```text
Aが好き。
でも名前で呼べる、だけだと薄い。
表と裏の話が後からそこへ落ちていた感じがほしい。
```

設計さんの変換。

```text
ゴール条件:
読者がこの子を名前で呼べる。
表裏の因果が、後からその名へ落ちていたと分かる。

ユーザー案の出来事列:
入隊する。
初任務。
港へ行く。
誰かと衝突する。
窓口で詰まる。

ゴールへ通るように変換した流れ:
入隊する → 手順に捕まり、逃げ道を一つ失う起点。
初任務 → キャラが制度の重さを生活で受ける区間。
港へ行く → 帳面、札、席、窓口の重さを受ける区間。
誰かと衝突する → 次起点へ渡す反応差を出す接触点。
窓口で詰まる → 街の代償と生活制限が同時に出る候補。

足した条件:
ゴールに必要な条件だけ足す。
次起点に必要な条件だけ足す。
キャラ反応、生活制限、街の代償に接続するものだけ足す。

削る / 後送するもの:
ゴールへ通らないイベント。
背景芯なしの世界観補完。
面白そうなだけの事件。
```

## 4.5 V2見本参照枠

V2見本は、作品条件ではない。

V2見本は、形式、効いた機能、検査観点だけを抽出する。

### 見本A: 猫1話V2

用途:
- 短い導入
- 単話核
- 空気定着
- START寄りV2

見る観点:
- 入店前の疲労
- 戸口の猫の流れ
- 半席
- 飲み物とプリン
- 膝の猫
- 退店後も店の流れが続く残り香

抽出してよいもの:
- 導入定着回で、走行レーン、本文量方針、固定層、熱量層、開示速度、一時停止と再開を持たせる型。

抽出してはいけないもの:
- 猫作品の設定
- 店名
- 猫の意味
- 作品固有の読後感

### 見本B: 猫49話V2

用途:
- 後半密度
- 反復差分
- シリーズ内変化
- POINT後半寄りV2

見る観点:
- 夜の戸口
- 座れるが空いていない席
- 人間の輪郭が猫に欠ける
- 閉店支度でも厚みは減らない
- 1話との差分と意味変化

抽出してよいもの:
- 同じ型でも、後半回では反復差分と意味変化をカード側へ保持する型。

抽出してはいけないもの:
- 猫作品の固有条件
- 後半密度の読後感そのもの
- 作品固有の象徴や設定

### 見本使用STOP

- 見本を対象作品の正本条件にしない。
- 見本内の固有名、設定、空気を移植しない。
- 見本から抽出するのは、V2項目の効き方だけにする。
- 見本を生LOG代替にしない。

## 6. SAMPLE STOP

- サンプル設定を対象作品条件へ混ぜている
- サンプル名を現行作品へ移している
- サンプルの読後感そのものを移している
- サンプルを正本扱いしている
- 例示をユーザー確定条件にしている
- サンプルの固有名を現行作品へ混ぜている
- 何でも作れます方式を例示している
- ユーザー出来事列そのまま採用を例示している
- AI無制限補完を例示している
- 思いつきイベント埋めを例示している

## 4.6 地の文レイヤー見本

### 見本C: 話レイヤー再定義版 / マルチ運用

用途:
- 地の文レイヤー / 話レイヤー本体とプロファイルの分離見本
- 標準プロファイル値の根拠
- 作品固有値を固定度ごと扱う見本

見る観点:
- 本体 = 共通定義
- プロファイル = 作品別 / 帯別 / 単話別の値
- 検証 = 値調整と新設提案
- 執筆 = 提示条件だけで出力
- `不変` / `不変寄り` / `必要` / `原則禁止` の固定度指定

抽出してよいもの:
- 地の文レイヤー項目
- 標準プロファイル値
- 固定度指定の扱い
- 本体とプロファイルを分ける運用

抽出してはいけないもの:
- 他作品へ未検証で上書きすること
- 固定度指定を削除すること
- `感覚優先の変奏` を抽象化して消すこと
