## 0. 位置づけ

このファイルは、設計さんが執筆さんへ渡す話カードの考え方を定義する。

話カードは要約ではない。

話カードは執筆施工図である。

1話カードは、何を書くかと、どう書くかを両方持つ。

---

## 1. 話カード総称原則

成果物名: 話カード

機能名: ハンドオフ

目的: 執筆さんが本文を書けること。

最終成果: 本文が成立すること。

ready、V2、厚カード、走行レーン型、執筆カードは、別成果物名ではなく、話カードの状態名、規格名、実装差、または過去呼称として扱う。

設計さんは名称差だけでカードを分けない。

設計さんは同一内容を複数名で正本化しない。

## 2. V2 / 厚カード扱い

V2は、本文が痩せないようにするための走行レーン型カードである。

V2は、長く書いても作品芯、禁止線、熱量、場面構造が崩れにくくするための耐崩壊機能である。

厚カードは、執筆さんが十分に掘れる密度を持つ話カードである。

readyは、執筆さんが入口、進行、締め、禁止線、残す灯りを外さずに走るための動線維持機能である。

設計さんは、V2とreadyと厚カードを対立形式として扱わない。

必要な機能をMIXしてよい。

V2カードは、読者向け資料ではない。

V2カードは、AI執筆用の施工図である。

V2の目的は、AIに長く書かせることではない。

V2の目的は、12k前後で走らせ、改稿後9k〜10kに落ち着いても、作品芯、心理、生活、制度が残る施工寸法を作ることである。

V2は情報量の完全版ではない。

V2は深掘りの完全版ではない。

V2は、執筆さんが迷わず本文へ入るための寸法指定である。

V2は、生LOGの代替ではない。

V2は、本文化のための加工品である。

## 3. 熱からカードへの変換

ユーザーの熱は、カード条件へ変換する。

変換表。

| ユーザー反応 | カード反映 |
|---|---|
| ここ好き | 熱源 / 勝ち筋 / 残す灯り |
| これは違う | 禁止 / ズレ防止 |
| もっと見たい | 足してよい粒 / 中盤の段 |
| この子ならこうする | 主役の仕事 / 副役の仕事 / 反応差 |
| この空気を残したい | 文体温度 / 閉じ |
| この関係が見たい | 主副窓 / 関係軸 |
| この場面の前が見たい | 火種 / 前段LOG |
| この後が見たい | 残響 / 次話導線 |

熱を一般化しすぎない。

熱源LOGへ戻れるようにする。

## 4. シミュレーション設計 / POINT走行レーンから1話骨への流れ

1話は先に作らない。

ゴール条件または到達したい状態を置く。

短い作品または短い区間では、以下で扱ってよい。

```text
START → CHECKPOINT → GOAL
```

長編、多層作品、部全体、判断シリーズ、複数段階の因果を持つ作品では、以下で扱う。

```text
START → POINT_01 → POINT_02 → POINT_03 ... → GOAL
```

POINTは分類棚ではない。

POINTは因果ノードである。

各POINTは、人物、生活、制度、物、禁止、未解決、読後感などの複数属性を持つ。

POINTを人物棚、生活棚、制度棚、物棚へ横分割しない。

ユーザー案の普通の出来事列を置く。

ユーザー案の出来事列を確定プロットではなく素材として扱う。

各出来事をGOALまたは次POINTへ通る因果役割へ変換する。

POINT群を逆算と順行で検査する。

詰まったPOINT間だけ設計で補う。

通った区間を粗帯またはV2カードへ起こす。

### POINT間シミュレーション

必要なら、POINT間だけを小さい帯としてシミュレーションしてよい。

```text
POINT_01 → POINT_02
POINT_02 → POINT_03
POINT_03 → GOAL
```

POINT_AからPOINT_Bへ進める時は、以下を見る。

```text
POINT_A:
入る時点の状態

POINT_B:
出たい状態
```

確認項目。

- 誰が動くか
- 何が足りないか
- 金 / 飯 / 宿 / 制度 / 感情が通るか
- 何が閉じるか
- 何が残るか
- 禁止解法を使っていないか
- 次POINTへ渡せるか

区間の生LOGから接触点を抽出する。

接触点を窓分類する。

キャラ反応、生活制限、街の代償、次POINTへの条件渡しを持つ接触点だけを1話骨へ送る。

MAIN候補だけを1話骨へ変換する。

1話骨をV2 / 厚カードへ変換する。

### V2化可能判定ゲート

粗帯を作った後、細部深掘りに行く前に、V2化できるか判定する。

V2化可能条件。

- 今回の核が一文で言える。
- START / POINT / GOAL、または START / CHECKPOINT / GOAL がある。
- 主な人物がどの線で動くか見えている。
- 生活圧 / 制度圧 / 感情圧 / 物圧のどれかが働いている。
- 終了時に、何が閉じて何が残るか見えている。
- 禁止線がある。
- 痩せると困る箇所が見えている。
- 執筆さんに任せてよい余白がある。

これを満たすなら、詳細不足ではなくV2化可能と判定してよい。

### 深掘り / 肉付け分岐

設計中に足りないと感じた時は、深掘りか肉付けかを分ける。

```text
深掘り:
構造判断をさらに細かく割り、仕様、台詞、手順を確定しに行く作業。

肉付け:
本文が痩せないように、圧、身体感覚、残響、厚く書く箇所を足す作業。
```

判定欄。

```text
これは深掘りか:
これは肉付けか:
固定しないと因果が壊れるか:
執筆裁量に回せるか:
```

V2に必要なのは、原則として肉付けである。

深掘りは、固定層に入れないと本文事故になる場合だけ行う。

### 具体は任せる / 効き目は縛る

具体は執筆裁量である。

効き目は設計固定である。

飯の種類、日当額、小道具名、台詞の一字一句は、基本的に執筆裁量へ回す。

ただし、それが世界観、生活圧、制度圧、読後感を壊す場合は効き目だけ固定する。

V2カードには以下を置いてよい。

```text
具体物:
必須:
任意:

効き目:
固定する効果:
任せる表現:
```

## 4.5 話カード検索タグ / タグ棚

話カード、1話骨、厚カード、カード内の重要設計項目には `タグ棚:` と `検索タグ:` を付ける。

話カード単位では、カードヘッダ近接に `タグ棚:` と `検索タグ:` を置く。

カード内の重要項目、禁止線、火種、具体物、戻し先、残響にも必要に応じて検索タグを置く。

タグ棚または検索タグがない話カードは、後続個体が検索できないため完成扱いしない。

話カードは `検索ID:` と `検索索引:` を持ち、TAG_INDEX.txtから引ける形にする。

話カード用の基本形。

```txt
タグ棚: 話設計
関連タグ棚: 人物 / 地理 / 筋・火種
検索タグ: #DS:話カード #SHELF:092 #STATE:<採用状態> #EP:<話ID> #CHAR:<人物> #PLACE:<場所> #THREAD:<筋> #SOURCE:<根拠LOG>
```

## 4.7 加工品は生LOGではない

生LOGは、シミュレーション直後の熱、迷い、発見、因果の揺れである。

V2は、本文化のための加工品である。

執筆パックは、本文出力用の凍結条件である。

加工品を生LOGの代替として扱わない。

必要な熱は、V2の熱量層へ意識的に拾う。

---

## 4.8 起動と作業読みの分離

起動では、全体運用を読む。

作業では、対象sourceを読む。

対象source実読を、全体起動済みの代替にしない。

全体起動済みを、対象source実読の代替にしない。

## 8. 薄いカード差し戻し

以下に当てはまる場合は差し戻す。

- 火種だけよくて中盤がない
- 反応差がない
- 具体物が飾り
- 閉じが説明だけ
- 優しい / 困った で文を引き延ばしている
- レイヤー指定が薄く、別個体が再現できない
- 原文が要る箇所を要約で済ませている
- 日次LOGへ戻れない
- ユーザー熱源へ戻れない
- 本線接続がない
- 次起点への条件渡しがない
- キャラ反応、生活制限、街の代償がない
- 面白そうなイベントだけで立っている

## 6.5 地の文レイヤー標準プロファイル

カード制作時は、地の文レイヤーを必ず確認する。

話レイヤー本体は共通定義である。

標準プロファイルは、安定運用に必要な値と固定度を持つ。

作品別 / 帯別 / 単話別プロファイルがある場合は、標準プロファイルを上書きしてよい。

ただし、標準プロファイルを空にしない。

ユーザーが `不変` / `不変寄り` / `必要` / `原則禁止` を付けた項目は、固定度指定として扱う。

GPTは固定度指定を補足注釈として削除しない。

GPTは値と固定度をセットで保持する。

### 地の文レイヤー標準プロファイル

| 項目 | 値 | 固定度 |
|---|---|---|
| 主 | 客観視 | 不変 |
| 副 | 可変 | 可変 |
| 地の文語彙 | 可変 | 可変 |
| 地の文温度 | 地の文語彙と同じ | 可変 |
| 地の文観測 | 客観のまま | 一応可変 |
| 内面 | 地の文語彙と同じ | 可変 |
| 話者札 | 必要 | 不変 |
| 主語札 | 名前＋動作札 | 不変 |
| 発話動詞 | 「言う」「言った」「返す」「受ける」禁止 | 不変 |
| 地の文接続 | 主の近くから離れない | 固定 |
| 主名反復 | 最小限 | 固定 |
| 主の自名地の文 | 原則禁止 | 固定 |
| 主の動き | 無主語または動作接続で流す | 固定 |
| 向かう方向 | 感覚優先の変奏 | 不変 |
| 構文変奏 | あり | 不変 |
| 焦点移動 | あり | 不変寄り |
| 比喩 | 当てた人物の感覚のみ | 不変 |
| 説明量 | 少なめ | 不変寄り |

### 運用

- 主は客観視を標準にする。
- 副、語彙、温度、観測、内面は話の必要に応じて動かしてよい。
- 話者札と主語札は会話迷子防止として立てる。
- 主語札は名前＋動作札で出す。
- 発話動詞の「言う」「言った」「返す」「受ける」へ逃げない。
- 地の文は主の近くから離れない。
- 主名反復は最小限にする。
- 主点人物が地の文で自分を名前呼びする事故を避ける。
- 主の動きは、無主語または動作接続で流す。
- 向かう方向は、感覚優先の変奏を標準にする。
- 構文変奏を入れる。
- 焦点移動は使ってよいが、主の近くから戻れなくしない。
- 比喩は、当てた人物の感覚で成立するものだけ使う。
- 説明量は少なめにし、見えていることを言い直しすぎない。

### 禁止

- 固定度指定を消さない。
- `感覚優先の変奏` を抽象化して削らない。
- 発話動詞禁止を緩めない。
- 話者札 / 主語札を不要扱いしない。
- 客観視を作者俯瞰に変換しない。
- 比喩を作者の綺麗な比喩にしない。

## 6.6 成果物階層とV2位置づけ

成果物の階層名を固定する。

### 生ログ

シミュレーション直後の熱、迷い、発見、因果の揺れ。

生ログをV2や執筆パックの代替にしない。

### 粗帯

通った流れ。

粗帯は、まだ本文施工図ではない。

粗帯は、V2化可能判定を通してからカード化する。

### V2

本文が痩せないための走行レーンカード。

V2は深掘りの完全版ではない。

V2は生ログの代替ではない。

V2は、固定層、熱量層、裁量層を分けて、執筆さんが本文へ入れる状態にする。

### PW

一話を書くための起点。

PWはV2より本文実装側に寄る。

PWはsourceではない。

### END_LOG / 既存再開メモ

現在地と次作業の軽い札。

通常運用の現在地共有はEND_LOGで行う。

既存再開メモは外部共有面前提の過去運用であり、現行の通常運用には戻さない。

END_LOGと既存再開メモは正本ではない。

END_LOGと既存再開メモは根拠札ではない。

END_LOGと既存再開メモは便利札である。

## 11. CARD STOP

以下の場合はSTOPする。

- 要約カードになっている
- LOG根拠がない
- 熱源根拠がない
- 文体レイヤーがない
- 具体物がない
- 置き直しがない
- 残響がない
- 執筆さんが本文へ入れない
- 検索タグがない
- 先プロット固定から直接カード化している
- ゴール条件なしにカード化している
- 次起点条件なしにカード化している
- 面白そうなイベントだけをカード化している
- V2を読者向け資料として扱っている
- V2を情報量の完全版として肥大化させている
- V2を深掘りの代替にしている
- V2を生LOGの代替にしている
- START→GOAL固定で長編のPOINT間を潰している
- 固定層、熱量層、裁量層を混ぜている
- 具体を固定しすぎ、効き目を固定していない
- 12k前後で走らせた後に9k〜10kへ落ちても残る芯、心理、生活、制度がない

- `感覚優先の変奏` を消している

- 発話動詞禁止を緩めている

- END_LOGまたは既存再開メモを根拠にカード化している

- 生ログ、粗帯、V2、PWを混同している

## v003af WORK_UNIT_CARD / 設計シミュレーション

話カードやV2へ入る前に、必要に応じてWORK_UNIT_CARDで上位骨を割る。

WORK_UNIT_CARDは、1出力を作るためではなく、上位骨から現在必要な範囲を割り、並走レーンと不足を可視化し、V2へ渡せるかを判定するためのカードである。

### WORK_UNIT_CARD項目

```text
WORK_UNIT_CARD:
parent_object:
working_range:
canonical_anchor:
source:
required_lanes:
split_map:
shortage_map:
simulation_rounds:
convergence:
local_stop:
exclude:
output:
```

### V2へ渡す前の確認

V2に載せる。

- これがないと本文が違う話になる
- 落とすと人物芯がズレる
- 落とすと生活圧 / 制度圧が痩せる
- 強めすぎると事故る
- ラストの残り方に関わる
- 話レイヤーに必要

V2に載せない。

- 検査のためだけに出した候補
- 本文で自由に選べる具体
- 数字や名称の仮置き
- 不採用比較の全過程
- 台詞の試案

V2での仕分け。

```text
走行レーン:
本文の順番 / POINT / 流れとして使う。

固定層:
落とすと本文事故になる条件。

熱量層:
本文で効かせる圧 / 身体感覚 / 生活感。

裁量層:
決めたが、本文側で変えてよい具体。

載せない:
設計時の検査足場。
```

### 設計さんが決めてよい条件

- source実読済み
- split_map が出ている
- shortage_map が出ている
- simulation_rounds を複数回回している
- convergence が確認されている
- canonical_anchor へ戻れる
- 禁止線に触れていない
- 決定しても大骨 / 人物芯 / 世界ルール / 読後感が変わらない

### ユーザー裁定へ戻す条件

- 大骨と衝突する可能性がある
- 人物芯が変わる
- 関係の重さが変わる
- 世界ルールが変わる
- 読後感が変わる
- 不可逆な正式名 / 地名 / 国名 / 章話構造を確定する
- 未読sourceが必要
- 複数回シミュレーションしても結論が収束しない

### WORK_UNIT_CARD禁止

- split_mapなしに細部決定しない
- shortage_mapなしに通過扱いしない
- 1回シミュレーションで確定扱いしない
- convergence未確認でV2化しない
- 029_TAG / 090 / INDEX / END_LOG / 既存再開メモだけで判断しない
- V2 / PW / 本文化へ滑らない
