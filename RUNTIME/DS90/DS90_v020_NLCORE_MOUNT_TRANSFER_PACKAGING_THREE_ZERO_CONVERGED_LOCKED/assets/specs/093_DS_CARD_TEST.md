検証は本文成果、候補反応、LOG保持、カード密度で見る。

## 4. 見せる量検証

見せる量は固定しない。

段階。

- 0: 本文成果だけをユーザーに見せる。
  - 本文成果は執筆さん、別レーン、またはテスト出力によるもの。
  - 設計さんが通常作業として本文執筆を担当する意味ではない。
- 1: 候補だけ見せる。
- 2: 候補と好き / 違うの選択を取る。
- 3: ゴール、起点、次起点、候補、窓分類を見せる。
- 4: プロット化表、不足修正まで見せる。
- 5: 厚カードまで見せてから執筆へ渡す。

初心者には本文成果だけを標準にしない。

慣れたユーザーには見せる量を減らしてよい。

段階0は、検証用または比較用の例外である。

段階0を初心者導線の標準にしない。

段階0を使う場合も、設計さん内部ではLOG、候補、分類、プロット化、厚カード化を省略しない。

本文成果だけを見せて、ユーザー反応、熱源LOG、採否分類を取らない運用は禁止する。

設計さん内部では、見せる量を減らしてもGOAL_TO_ANCHOR_FLOW、LOG、候補、分類、プロット化、厚カード化を省略しない。

## 6. 既知札

### 既知札 001

先プロット型は、慣れたユーザーには速い。

注意:
初心者には何を決めればよいか見えにくい。

### 既知札 002

熱提示型は、初心者が好き / 違う / もっとで反応しやすい。

注意:
候補を出しすぎると散る。

### 既知札 003

GOAL_TO_ANCHOR型は、初心者の熱源を散らさず、話の増殖を抑えやすい。

注意:
ゴール条件と次起点条件が薄いと、区間シミュレーションがイベント発明へ戻る。

### 既知札 003.5

普通の出来事列変換型は、初心者が出しやすい。

注意:
ユーザーの出来事列を確定プロットとして扱うと、薄いイベント列へ戻る。

設計さんは、出来事をゴールへ通る因果役割へ変換してからGOAL_TO_ANCHORへ送る。

### 既知札 004

厚カード型は本文量と再現性に強い。

注意:
LOG根拠がないと要約カード化する。

### 既知札 005

日次シミュレーション型は、起点間の下位処理として有効である。

注意:
標準手順そのものとして固定すると、候補爆増、目的地の薄さ、話の横滑りが起きる。

## 7. CARD TEST STOP

- 本文成果だけ見てユーザー反応を見ていない
- 熱源LOGを無視している
- LOG根拠なしにカード評価している
- 形式名だけで採用している
- 初心者負荷を確認していない
- 見せる量を固定正解にしている
- ゴール条件なしに比較している
- 次起点条件なしに話候補を評価している
- ユーザー案の出来事列を確定プロット化している
- 背景芯なしにAIが世界観を確定している
- 面白そうなイベントだけを合格にしている

## v003af V2走行レーン検査

検査対象:
- 粗帯
- V2カード
- 厚カード
- 執筆依頼文

検査項目:
- START→CHECKPOINT→GOAL固定で長編を潰していない
- 必要に応じてPOINT群を置いている
- POINTを分類棚ではなく因果ノードとして扱っている
- POINT間で誰が動くか、何が足りないか、何が閉じて何が残るかを見ている
- 通ったPOINT間だけをV2化している
- V2が深掘りではなく、本文が痩せないための肉付けカードになっている
- 固定層 / 熱量層 / 裁量層が分かれている
- 痩せると困る箇所、勝手に強めない種、開示速度、反復チェック、ラストの残り香がある
- 執筆依頼文で本文へ入れる

## v003ap WORK_UNIT_CARD検査

検査対象:
- WORK_UNIT_CARD
- POINT群
- 並走レーン
- 不足一覧
- 収束確認
- V2へ渡す層仕分け

検査項目:
- 1作業単位を1出力に閉じていない
- parent_object が見えている
- working_range が閉じている
- canonical_anchor がある
- sourceを実読している
- 090 / 029_TAG / INDEX / END_LOG / 既存再開メモだけで作品判断していない
- required_lanes を並走で見ている
- split_map がある
- shortage_map がある
- simulation_rounds を複数回回している
- convergence を確認している
- 割れた先が canonical_anchor へ戻れる
- 不足があるのに補完で進めていない
- 設計さんの自動決定が、source実読 / split_map / shortage_map / convergence / canonical_anchor / 禁止線確認の範囲に収まっている
- ユーザー裁定へ戻す条件が、細部ではなく芯変更に限定されている
- V2へ渡すもの / 渡さないものが分かれている
- V2 / PW / 本文化へ滑っていない

比較時の注意:
- `objectが複数` だけで失格にしない。
- `canonicalが割れる` だけで失格にしない。
- 骨が割れた後に上位骨へ戻れるかを見る。
