| 削除対象 | 理由 | 代替 | 判定 |

| 共創モード | モード化により状態切替が増える | OUTPUT型 / 委任範囲 / STOP | 削除 |

| 発明モード | AI補完を正本化しやすい | 候補生成 / 採否分類 | 削除 |
| 作業モード | 曖昧 | OUTPUT型 | 削除 |
| 先に細かいプロットを固定する | 初心者が詰まる / LOG由来の話を潰す | 骨先行 / シミュレーション後プロット化 | 削除 |

| 最初に設定表を埋めさせる | 知識と技術を要求する | 候補提示 / 熱選択 | 削除 |
| 質問攻めで条件回収 | 初心者が答えられない | 見せて選ばせる | 削除 |
| 何でも作れます方式 | 話が無限に増えて熱源が溶ける | GOAL_TO_ANCHOR_FLOW | 削除 |
| 思いつきイベントで間を埋める | 次起点条件へ接続せず、候補だけ増える | 起点A→起点Bの条件渡し | 削除 |

| AIが足りない出来事を無制限に足す | 作品外の世界観を勝手に確定し、熱源からズレる | 足してよい条件の限定 / 未確認分類 | 削除 |
| 日次シミュレーションを標準手順そのものにする | DAY単位が目的化し、ゴールと次起点が薄くなる | 起点間シミュレーションの下位処理 | 退役 |
| 固定モード名で深さを切り替える | モード人格化し、挙動が割れる | 出力深度 / 区間幅 / 確認頻度の調整 | 削除 |
| AI本文成果だけ提示 | 内部工程が見えない | 見せる量制御 / 検証用例外 | 削除 |
| 028へ引継ぎ全部 | 棚責務混線 | 021/022/024/028/099振り分け | 削除 |
| 095に作品引継ぎ本文蓄積 | 090帯が作品棚化する | マウントZIP出力制御 | 削除 |
| 仮到達点一覧を話数表へ誤変換 | 生活反復、心理変化、制度処理が落ちる | 長大アーク管制 / 到達判定LOG | 削除 |
| LOGなし話数案 | 圧縮事故を検出できない | LOG後の話数仮算出 / 再検査 | 削除 |
| 話数仮算出を完成値扱い | 到達範囲が未確認のまま固定される | 到達範囲確認 / 結果安定判定 | 削除 |
| 処理限界超えの成果物風出力 | 薄い案を完成案に見せる | 処理単位分割 / 限界STOP文 | 削除 |
