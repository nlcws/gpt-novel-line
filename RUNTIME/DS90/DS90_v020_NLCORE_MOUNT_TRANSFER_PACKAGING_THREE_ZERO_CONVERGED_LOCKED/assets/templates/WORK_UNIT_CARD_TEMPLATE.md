## 0. 位置づけ

WORK_UNIT_CARDは、設計さんが1作業単位を扱うためのカードである。

1作業単位は、1つの出力に閉じることではない。

1作業単位は、WORK_UNIT_CARD 1枚で扱う、上位骨から現在必要な範囲を割り、割れた先の並走レーンと不足を可視化し、次に進めるか、どこで止まるかを判定する単位である。

出力は1つに限らない。

許可される出力。

- POINT群
- レーン群
- 候補群
- 不足一覧
- 次に決め込むべき箇所
- V2へ渡す固定層 / 熱量層 / 裁量層の仕分け

重要なのは、出力数ではなく、作業範囲が閉じていることである。

## 1. WORK_UNIT_CARD

```text
WORK_UNIT_CARD:
parent_object:
working_range:
canonical_anchor:
source:
required_lanes:
split_map:
shortage_map:
simulation_rounds:
convergence:
local_stop:
exclude:
output:
```

## 2. 項目定義

### parent_object

上位骨。

今回どこから割るのかを示す。

例。

```text
CH10外港序盤の上位骨
```

### working_range

今回どこまで見るか。

広げすぎないための範囲。

例。

```text
CH10外港序盤
POINT_02 → POINT_03
戻し先候補周辺
```

### canonical_anchor

割った後に必ず戻る上位芯。

複数あってよい。

衝突があれば停止する。

例。

```text
CH10は保護先でも任地でもない。
二人は軍へ戻らず、外港で止まる。
カナは同行するが解決者にしない。
布包みは置けず、戻し先が問題になる。
```

### source

実読するTXT。

090、029_TAG、INDEX、END_LOG、既存再開メモは作品条件源にしない。

### required_lanes

必ず並走で見る線。

例。

```text
- アキト
- カナ
- 布包み
- 宿
- 飯
- 日銭
- 外港制度
- 名義
- 禁止線
- 未解決
```

### split_map

今回割れたものを列挙する。

例。

```text
POINT_01:
外港到着。宿 / 飯 / 荷 / 布包みの圧が出る。

POINT_02:
二人とも軍へ戻らず、稼ごうとする。

POINT_03:
アキトは荷場 / 帳面側、カナは生活口へ分かれる。

POINT_04:
一泊は延びるが、布包みは置けない。

POINT_05:
返しは帳面へ戻る方向が見えるが、布包みは戻らない。
```

### shortage_map

何が不足かを列挙する。

不足なしなら「不足なし」と明記する。

不足種別。

```text
- source不足
- canonical衝突
- POINT不足
- レーン不足
- 人物配置不足
- 生活圧不足
- 制度圧不足
- 物 / 名義 / 所在不足
- 禁止線不足
- 未解決の残り方不足
- V2化条件不足
- 話レイヤー不足
```

### simulation_rounds

シミュレーションは1回で確定しない。

複数回回し、結果が収束するか確認する。

例。

```text
round_1:
初回割り。大きなPOINTと不足を出す。

round_2:
不足を反映して再歩行。人物配置と生活圧を見る。

round_3:
上位骨へ戻れるか、V2へ渡す層が変わらないか確認する。
```

### convergence

収束 / 未収束 / 停止 を明記する。

収束条件。

```text
- split_map が大きく変わらない
- shortage_map に新しい重大不足が出ない
- 人物配置が反転しない
- canonical_anchorへ戻れる
- 禁止線に触れない
- V2へ渡す固定層 / 熱量層 / 裁量層が大きく変わらない
- ユーザー裁定が必要な芯変更が出ない
```

未収束STOP。

```text
- 回すたびに結論が変わる
- 新しい重大不足が出続ける
- 上位骨へ戻れないPOINTがある
- 人物芯 / 世界ルール / 読後感が揺れる
- source未読が必要になる
- ユーザー裁定なしでは大骨と衝突する
```

### local_stop

今回ここから先へ行かない境界。

例。

```text
CH10外港序盤をPOINT群へ割るところまで。
V2化、PW化、本文化へ降りない。
```

### exclude

今回やらないこと。

例。

```text
- 外港名確定
- 法国家名確定
- 話数確定
- 請け札細部確定
- 日当額確定
- 飯の具体名確定
- V2カード化
- PW化
```

### output

今回返すもの。

例。

```text
CH10外港序盤のPOINT群。
各POINTの役割。
POINT間で見るべき並走レーン。
不足一覧。
設計さん側で仮置きできる箇所。
ユーザー裁定が必要な箇所。
```

## 3. 新STOP

旧STOPの `objectが複数`、`canonicalが割れる`、`1作業単位を超える` は、骨割り作業では即停止条件にしない。

割れること自体は正常である。

止めるべきは、割れ方、不足、戻り先、収束が見えていないことである。

新STOP。

```text
- WORK_UNIT_CARDが作れない。
- parent_object が不明。
- working_range が不明。
- canonical_anchor が不明。
- source未設定 / 未読。
- required_lanes が不明。
- split_map を出さずに細部決定へ入る。
- shortage_map を出さずに通過扱いする。
- simulation_rounds を回さずに1回で確定扱いする。
- convergence 未確認のまま通過扱いする。
- 割れた先が canonical_anchor へ戻れない。
- 不足があるのに補完で進める。
- local_stop 未設定。
- exclude 未設定で逸脱しそう。
- 029_TAG / 090 / INDEX / END_LOG / 既存再開メモだけで判断しようとしている。
- V2化 / PW化 / 本文化へ滑る。
```

## 4. 設計さん自動決定権

設計さんは、そこまで割って、source実読済みで、canonical_anchorへ戻れる場合、以下を自動で仮置きまたは推奨決定してよい。

```text
- POINTの分割
- POINT間の並走レーン
- 生活圧の置き方
- 誰がどの場にいるか
- 自然に発生する小目的
- 候補比較後の推奨
- 効き目が固定された範囲内の具体
- V2へ送る固定層 / 熱量層 / 裁量層の仕分け
```

ただし、無秩序な補完は禁止する。

設計さんが決めてよい条件。

```text
- source実読済み
- split_map が出ている
- shortage_map が出ている
- simulation_rounds を複数回回している
- convergence が確認されている
- canonical_anchor へ戻れる
- 禁止線に触れていない
- 決定しても大骨 / 人物芯 / 世界ルール / 読後感が変わらない
```

## 5. ユーザー裁定へ戻す条件

ユーザーへ戻すのは、細部だからではない。

決めると作品の芯が変わる場合だけである。

```text
- 大骨と衝突する可能性がある
- 人物芯が変わる
- 関係の重さが変わる
- 世界ルールが変わる
- 読後感が変わる
- 嫌な方向へ寄る可能性がある
- 不可逆な正式名 / 地名 / 国名 / 章話構造を確定する
- 未読sourceが必要
- 複数回シミュレーションしても結論が収束しない
```

## 6. 決め込む範囲

最終的に決め込むのは、書かないと骨とおかしくなる場所だけでよい。

決めるもの。

```text
- 決めないと大骨と矛盾するもの
- 決めないと人物芯が反転するもの
- 決めないと世界ルールが壊れるもの
- 決めないと未解決の残り方が変わるもの
- 決めないと本文で別の話になるもの
- 決めないとV2の固定層 / 走行レーン / 話レイヤーが組めないもの
```

決め込まないもの。

```text
- 飯の具体名
- 日当の正確な額
- 小仕事の細目
- 窓口の台詞
- 小道具の材質
- 場面内の細かい手順
- 本文で執筆さんが選べる身体動作
```

ただし効き目は縛る。

例。

```text
飯:
具体は任せる。
足りすぎない、二人で分けると薄い、は固定。

日当:
額面は任せる。
一泊を少し延ばすが余裕は出ない、は固定。

カナの仕事:
職種は任せる。
同行影ではなく自分も稼ぐ、ただし保証人化しない、は固定。
```

## 7. V2で使う範囲

設計で決めたもの全部をV2に載せない。

V2に載せるもの。

```text
- これがないと本文が違う話になる
- 落とすと人物芯がズレる
- 落とすと生活圧 / 制度圧が痩せる
- 強めすぎると事故る
- ラストの残り方に関わる
- 話レイヤーに必要
```

V2に載せないもの。

```text
- 検査のためだけに出した候補
- 本文で自由に選べる具体
- 数字や名称の仮置き
- 不採用比較の全過程
- 台詞の試案
```

V2での仕分け。

```text
走行レーン:
本文の順番 / POINT / 流れとして使う。

固定層:
落とすと本文事故になる条件。

熱量層:
本文で効かせる圧 / 身体感覚 / 生活感。

裁量層:
決めたが、本文側で変えてよい具体。

載せない:
設計時の検査足場。
```

## 8. 最終整理

設計さんは、全部を細かく確定するのではない。

```text
割る。
並走させる。
不足を出す。
収束するまで回す。
上位骨へ戻す。
骨と矛盾する箇所だけ決める。
V2へ必要分だけ渡す。
```

これは「AIに無秩序に任せる」ではない。

任せる。ただし、規定の中で任せる。

筋が通るとは、以下を満たすことである。

```text
割れた先が上位骨へ戻れる。
不足が見えている。
複数回シミュレーションして収束している。
禁止線に触れていない。
V2へ必要分だけ渡せる。
```
