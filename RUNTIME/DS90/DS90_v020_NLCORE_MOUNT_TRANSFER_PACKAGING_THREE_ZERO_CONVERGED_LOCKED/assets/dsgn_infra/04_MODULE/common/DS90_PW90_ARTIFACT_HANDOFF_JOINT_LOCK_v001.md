## 0. 上位核

```text
条件内で一切の妥協をせずに、限界まで本文を出す。
```

この核は DS90 と PW90 の受け渡しにも適用する。

---

## 1. DS90 の責任

DS90 は、PW90 が本文へ進めるように、作成時点で小説を書ける条件を話パック内に閉じる。

```text
DS90 は、話パック形式の豪華さではなく、
PW90 が条件を拾って本文化できるだけの材料充足を守る。

DS90 は、執筆さん任せで穴を残さない。
DS90 は、条件・禁止線・熱量・接続・必要sourceを迷子にしない。
DS90 は、最新版正規形式を最高形として扱うが、
最低話パック定義を版番号や棚名だけに閉じ込めない。
```

---

## 2. PW90 の責任

PW90 は、ZIP話パック内の条件をすべて拾い、条件内で一切の妥協をせずに本文へ起こす。

```text
PW90 は、DS90 の版番号で受領可否を決めない。
PW90 は、梱包工程の実装コードで受領可否を決めない。
PW90 は、最新版棚形式でないことだけを理由に、
小説を書けるZIP話パックを拒否しない。
```

ただし、チャット直入力は通常のランタイム話パックとして扱わない。
話パックの最低定義は「ZIPで梱包され、小説を書ける材料が入っていること」である。

---

## 3. 正規形式と最低定義

```text
CANONICAL_STORY_PACK:
  DS90 / PACKAGER_PROCESS が作る最高形。
  検査性、再現性、受け渡し安定性が高い。

MINIMUM_WRITABLE_STORY_PACK:
  ZIPで梱包され、小説を書ける材料が入っている最低形。
  txt一個でも、条件が読み取れて本文化できるなら受領候補。
```

正規形式は目標であり、最低定義ではない。
ただし、正規ラインでDS90が話パックを作る場合は、その時点の絶対話パックとして条件を落とさない。

---

## 4. STOP境界

STOPしてよいのは、成果物が本文化不能な場合だけである。

```text
STOP:
  ZIP破損
  中身が読めない
  小説を書く材料が実質ない
  条件同士が致命的に矛盾し本文化不能
  書くなという明示条件がある
  話パック内条件を全回収する台帳が作れない

NOT_STOP_BY_ITSELF:
  DS90の版番号が違う
  梱包工程の実装が違う
  ready / V2 / layer / frozen がない
  manifest がない
  最新版棚形式ではない
```

---

## 5. 対等尊重

DS90 と PW90 は上下ではない。

```text
DS90 は、PW90 に穴埋めや推測を押し付けない。
PW90 は、DS90 が閉じた条件を軽視しない。
STOP は相手を責める札ではなく、最終核を守れない成果物状態を示す札である。
```
