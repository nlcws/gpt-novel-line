# PROJECT_HISTORY_SHELF_POLICY v019.15

STATUS: CURRENT_SPEC
APPLIES_TO_RUNTIME: v019.15-NLCORE-HISTORY-MASTER-REAPPLY-LOCKED

プロジェクト側棚は、履歴と思想を置く場所である。
正規ランタイムZIPは履歴を持たず、現行一本道だけを持つ。

## プロジェクト側棚へ置くもの

```text
なぜその思想になったか
何を捨てたか
何を禁止にしたか
どの判断が固定されたか
次の版上げで参照する思想ログ
```

## プロジェクト側棚へ置かないもの

```text
過去版ZIPそのもの
旧ランタイム一式
旧ファイル群の丸抱え
使わない実体コピー
```

## 固定

コード、棚構造、ファイル名、検査実装、テンプレ実体は可変。
思想、禁止線、責任境界、判断基準、工程目的は不変。

版上げ時は旧実装を移植しない。
旧実装が守っていた思想を読み、現行構造で再実装する。


## v019.15 master reapply rule

v019.15では、v019.14eをマスターとして扱い、履歴棚を削る方向の改修を母体にしない。
以後の改修は、履歴棚を戻すのではなく、削る前のマスターへ必要な改修をかけ直す。

## Active core and history shelf boundary

現行 active core は一本道に保つ。
ただし、プロジェクト側の履歴棚・思想棚・判断理由・事故ログ・版境界は捨てない。

通常実行では履歴で動かない。
マウント移管、棚更新、版上げ、差し替え判断、別個体引き継ぎでは履歴棚を参照する。
履歴棚を本文材料・話材・設定補完の源にしてはならない。
