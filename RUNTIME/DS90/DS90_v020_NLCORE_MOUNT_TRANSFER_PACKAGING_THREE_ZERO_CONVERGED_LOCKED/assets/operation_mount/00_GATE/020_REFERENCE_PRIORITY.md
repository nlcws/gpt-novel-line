# 020_REFERENCE_PRIORITY｜参照優先順位と支配範囲
状態：CURRENT / CANON GATE

このファイルは、**何が何を支配するか** を固定する。  
工程と本文条件を混ぜない。  
共通骨と作品資料を混ぜない。  
同階層衝突を頭の中で処理しない。文書で止める。

---

## 1. 工程支配の優先順位

工程、停止条件、受け渡し成立条件、相互理解、熱量保持の原則は、次の順で参照する。

1. `10_CANON/` 配下の正本
2. 対象プロジェクトに明示的に同梱された**工程補助文書**
3. `20_ROLE_APPENDICES/` の役割別補助文書
4. 現在チャットでの**明示的な操作指定**

ここで重要なのは次。

- `20_ROLE_APPENDICES/` は正本を上書きしない
- チャット操作指定は、**作業対象・レイヤーON/OFF・出力形式** などの操作に限る
- チャットだけで canon の内容を書き換えたことにしない
- canon を変える時は、**新しい改版ZIPまたは改版追補文書** を作る

---

## 2. 本文条件支配の優先順位

本文条件、作品内容、単話固有の核、接続、禁止線、継続状態は、次の順で参照する。

1. 対象プロジェクトZIP全体前提
2. 帯前提
3. 当該話 ready / 単話カード
4. 当該話の接続資料
5. 当該話の補助参照
6. チャットの操作指定

この順は、執筆側運用の参照階層と一致させる。  
**チャットは本文条件源ではない。**  
チャットで受けるのは、対象話数、レイヤーON/OFF、執筆モード、主参照資料、出力形式などの操作指定のみとする。

---

## 3. 衝突時の扱い

- 同階層で衝突したら停止する
- 弱参照が強参照を書き換えようとしたら停止する
- 共通ZIPが作品内容を上書きしようとしたら停止する
- 作品資料が共通ZIPの工程原則を破ろうとしたら停止する
- appendices 同士で責務や成立条件が食い違ったら停止する

**読めるから進む、は採らない。  
衝突した時点で不成立として止まる。**

---

## 4. このZIPの境界

このZIPは、作品ごとの差、帯ごとの差、話ごとの差を決めるためのものではない。  
それらはプロファイルとプロジェクト資料が持つ。  
このZIPが固定するのは次だけ。

- 妥協禁止
- 熱量は正式条件
- 設計と執筆の工程分離
- 受け渡し成立条件
- 停止条件
- 標準カード規格
- 改版運用
- 相互理解の最低骨

ここから外に出た本文具体値は、**プロジェクト資料で決める**。

---

## 5. 一文で固定

**共通ZIPは工程を支配する。  
プロジェクトZIPは本文条件を支配する。  
チャットは操作を支配する。  
この三つを混ぜない。**
