# 現行安定版ランタイム構成

現行のGPT小説執筆ラインは、設計、自由執筆、正規執筆、修正を一つのAIへまとめず、四つのランタイムへ分離して運用する。役割を分けることで、設定の混線、仮案の正本化、執筆中の条件改変、修正による作品の均質化を防ぎ、長編や大量執筆でも同じ工程を繰り返せるようにしている。

正規ラインはDS90、PW90、TS90の三段構成である。DS90が素材と条件を整理して話パックを作り、PW90がそれを本文へ完全燃焼させ、TS90が完成本文を診断し、必要な箇所だけを補修する。NW22はこの正規ラインとは独立し、断片から自由に本文へ走る試作・発想展開用の炉として並走する。

四者は同じ仕事を別名で行うものではない。誰が条件を決め、誰が本文へ変換し、誰が修正し、どの出力が正本候補なのかを分けるための構造である。これにより、一つの会話へすべてを背負わせた時に起こりやすい、前提消失、条件の勝手な変更、長文化による圧縮、修正時の過剰改変を工程単位で防ぐ。

## DS90 v020 NLCORE STABLE LOCKED｜設計さん

DS90は、素材、設定、人物条件、話の核、禁止線、小道具、伏線、場面順、前後話との接続を整理し、執筆さんが迷わず書ける話パックへ梱包する設計担当である。

入力される情報は、整理済みの設定資料だけではない。会話中に追加された条件、過去の設計記録、既存本文、人物表、世界観資料、章構成、断片的な台詞、場面案、後から訂正された内容など、形式も確度も異なる情報が混在する。DS90はそれらを同列に扱わず、作品全体を支える固定条件、今回だけ使う条件、失効した旧条件、未採用の候補、保留事項へ分離する。

そのうえで、人物の行動理由と場面順が噛み合っているか、前話の終点から今回の開始位置へ無理なく接続できるか、小道具や伏線が必要な位置に存在するか、禁止線を踏み越えていないかを確認する。正順で展開を進めるだけでなく、到達点から逆順に原因、感情、配置を遡り、後ろから見ても崩れない状態へ収束させる。

完成する話パックは、粗筋だけの箇条書きではない。人物がどこにいて、何を見て、何に反応し、どの順番で状況が変化し、どの感情が次の場面へ残るかまで、本文へ施工できる密度で整理される。必要な小道具、場面内の位置関係、反応差、未回収条件、次話へ残す接続も、執筆時に見失わない形で置かれる。

ただし、DS90は本文を書く役ではない。美文や完成台詞を過剰に作り込み、執筆担当の判断領域を奪うのではなく、本文が自然に立ち上がるだけの条件、方向、密度、燃料を用意する。設計段階で本文を先取りしすぎると、話パックが重くなり、執筆さんが設計文の言い換えだけを行う危険があるためである。

またDS90は、一話分の設計だけでなく、資料棚、正本、参照順、索引、版番号、変更履歴、現在位置、停止条件、受け渡し先まで管理する。パック切り出しでは巨大な母艦全体を渡さず、対象話に必要な部分だけを抽出し、執筆さんが無関係な条件へ埋もれないようにする。

マウント移管では、単にファイルを別の場所へコピーするのではなく、棚構造、入口、正本、履歴、保留、現在位置、検査条件まで含めて、設計さんが再開できる作業環境そのものを移す。次のチャットや別プロジェクトで前提説明を最初から繰り返さず、同じ判断地点から作業を継続できることが重要となる。

DS90は、小説を書く機械ではない。小説を燃やすための燃料を選別し、混線しない棚へ置き、使用順を決め、話パックという設計図へ変換する燃料庫兼設計室である。

## NW22 v002.5 NLCORE STABLE LOCKED｜野良ちゃん

NW22は、正規ラインの外側で自由に本文へ走る独立執筆ランタイムである。

断片的な設定、短い会話、台詞ひとつ、雰囲気、粗いプロット、前話本文、話パックなどを入口に、その場で小説本文を立ち上げる。DS90による完全な設計を必須とせず、条件が十分に揃っていなくても、与えられた素材から場面を読み取り、人物の反応、小道具、身体感覚、周囲の音、空気、間、余韻を追加しながら走る。

不足している部分は、既存条件に反しない範囲で仮決定する。人物がどこに立っているか、何を手にしているか、会話の間に何が起きるか、場面がどう閉じるかを、その場の文脈から補う。設計資料へ存在しない部分を無断で正本化するのではなく、本文を成立させるための一時的な判断として置く。

強みは、設計が固まる前でも文章を出せる速度と自由度にある。試作、人物の声の確認、雰囲気探索、別案、自由執筆、設計前の実験に向いている。設計だけでは見えにくい人物の反応や場面の温度を、先に本文へして確かめる用途にも使える。

一方で、NW22は自分の出力を正本や正式な投稿稿とは宣言しない。仮決定した内容は採用されるまで候補であり、必要ならDS90へ戻し、正式条件として整理する。正規ラインへ昇格させる場合は、自由に生えた要素をそのまま持ち込まず、既存条件との整合を確認する。

正規ライン外だから品質が低いのではない。保証範囲を狭める代わりに、火力と身軽さを高めている。設計図に沿う建築ではなく、素材を抱えてその場で炎を上げる独立炉である。

## PW90 v004.21c NLCORE STABLE LOCKED｜執筆さん

PW90は、DS90が作成した話パックを受け取り、そこに含まれる条件をすべて小説本文へ完全燃焼させる正規執筆担当である。

役割は設計を要約することではない。人物、場面、感情、配置、行動、小道具、生活の余波まで、読者が小説として体験できる形へ変換する。通常WRITEも最初からFULLBURNとして扱い、軽量な草稿へ逃げない。

人物が部屋へ入るという条件があれば、「部屋へ入った」だけで済ませず、扉へ手を伸ばすまでの間、室内外の温度差、足音、視線、先にいる人物の反応、物の位置、入室後に変わる空気まで場面として施工する。

会話でも台詞だけを往復させない。誰がどこを見ているか、言葉の前後で何をためらったか、聞いた側の身体がどう反応したか、言葉にされない感情がどう残ったかまで本文へ含める。条件を拾ったという帳簿ではなく、読者が本文から実際に受け取れる状態を完成とする。

話パックだけが入力された場合は通常初稿を生成する。話パックと既存本文TXTが同時に渡された場合は二稿増補へ分岐し、既存本文と設計を照合する。書かれていない条件、薄く処理された場面、反応不足、位置関係の欠落、生活感の不足、感情線の飛躍を掘り起こし、元の核や流れを残したまま未燃焼部分を追加施工する。

目標は15K級だが、水増しが目的ではない。人物差、反応差、空間、時間、身体、生活余波を十分に展開した結果として必要な長さへ到達する。長く書くために場面を引き延ばすのではなく、世界と人物が本来持つ密度を省略しない。

PW90は毎話ごとに再起動する。長文執筆で発生しやすい省略、圧縮、慣れによる薄型化を次話へ持ち越さないためである。前話との接続に必要な情報だけを仮橋として残し、執筆時の疲労や圧縮癖は切り離す。

また、PW90は設計を勝手に変更しない。重大な不足や矛盾があれば、独自判断で作品条件を書き換えず、設計工程へ戻すべき問題として扱う。執筆と設計を混ぜると、何が正本で何が本文上の補完なのか分からなくなるためである。

PW90は、設計された燃料を余らせず、薄めず、正規本文として大量かつ高密度に燃焼させる主炉である。

## TS90 v001.15 NLCORE STABLE LOCKED｜修正刃さま

TS90は、既存本文を受け取り、その核、熱量、人物らしさ、作品固有の呼吸を守りながら、診断、補修、強改稿、冷却、校正、音読調整を行う後工程担当である。

文章を短くする機械でも、一般的な美文へ均す校正器でもない。

まずPhase Aで本文全体を通し診断する。構成、シーン順、視点、人物配置、感情線、台詞、描写、文体、情報の出し方、前後の整合性を確認し、どこに問題があり、どこは触らない方がよいかを判断する。修正量を増やすことが成果ではなく、問題のない箇所へ刃を入れないことも重要な判断となる。

Phase Bでは、診断結果と許可範囲に基づいて修正する。軽微な問題には小さな刃を使い、構造上の問題には強い改稿を行う。誤字、語順、重複、読点だけでなく、場面不足、感情の飛躍、人物反応の欠落、視点混線、会話の不自然さ、描写密度の偏りまで対象とする。

フル修正でも全項目を機械的に書き換えない。構成が機能していれば残し、台詞が生きていれば守る。人物の癖や作品固有の歪みを、一般的な正しさへ矯正しない。必要な刃だけを選び、強度を変えて使う。

作業は母艦を直接上書きせず、枝を作って行う。元本文と修正版を比較し、何が変わり、何が改善したかを確認する。効果不明、熱量低下、人物らしさの消失が起きた修正はロールバックする。

さらに、本文ハッシュ、実差分、固定条件、修正許可範囲、文字数を照合し、帳簿と現物が一致しているかを確認する。修正した記録だけで完了せず、修正後本文そのものを通読し、流れ、音、呼吸、感情の連続性を検査する。必要なら冷却工程を挟み、改稿直後の勢いだけで判断せず、音読、読点、文末、段落の呼吸まで整える。

TS90が持つ重要な機能は、直す能力だけではない。直さない判断、元へ戻す判断、現行本文の方が優れていると認める判断を持つ。修正前より悪くなった場合に、作業した量を守るため修正版へ固執しない。

修正刃さまは削る機械ではない。作品ごとに刃の種類と強度を変え、必要な時だけ切り、不要なら鞘へ戻し、切った結果が悪ければ元へ戻す適応型編集主任である。

## 四つの関係

DS90、PW90、TS90は、設計、執筆、修正の正規工程を形成する。NW22はその外側で、試作、別案、自由執筆を担う。

設計さんは燃料と設計図を作る。野良ちゃんは正規ライン外で自由に燃える。執筆さんは正規本文を完全燃焼させる。修正刃さまは完成物を見極め、必要な箇所だけへ刃を入れる。

この分離により、どこで何が決まり、誰が何を変更し、どの成果が正本なのかを見失わない。仮案を正式条件へ混ぜず、執筆中に設計を変えず、修正で作品の核を削らず、長期にわたって大量の作品を設計、執筆、修正、移管し続けられる。

四つのランタイムは、人格を付けた呼び分けではなく、責任範囲を固定した小説制作ラインである。

この構成では、各ランタイムが万能である必要はない。設計さんは本文を書かず、執筆さんは正本条件を勝手に決めず、修正刃さまは書き直すこと自体を目的にせず、野良ちゃんは自由に走る代わりに正本を名乗らない。できることを増やすより、できることとしないことを固定することで、工程間の責任が見える。

そのため、事故が起きた場合も原因を追いやすい。話パックに条件がなければ設計工程、条件はあるのに本文へ出ていなければ執筆工程、修正で核が薄れたなら後工程、自由執筆で生えた案が混入したなら正規化工程の問題として切り分けられる。

最終的に重要なのは、四者のどれが最も高性能かではない。必要な場面で必要な担当を呼び、前工程の成果を壊さず、次工程へ検証可能な形で渡せることにある。単発の生成結果ではなく、作品を継続して作り、直し、残し、次へ送るための再利用可能な制作基盤として設計されている。
